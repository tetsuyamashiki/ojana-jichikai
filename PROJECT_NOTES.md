## 更新の流れ
1. Claudeがindex.html等をフォルダ内で直接編集
2. Claudeが `git add` → `git commit` → `git push` まで実行(2026-07-10からClaude側で完結)
   - Mac側で `git config --global credential.helper store` を設定し、
     `~/.git-credentials` にGitHub Personal Access Token(PAT)を保存済み
   - トークンの有効期限が切れた場合は、GitHubで再発行し、ユーザーが
     `~/.git-credentials` の内容を更新する必要がある(https://github.com/settings/tokens)
   - 旧ワークフロー(ユーザー自身がターミナルでpush)も引き続き利用可能
3. 大きな更新時はSearch Consoleで「URL検査→インデックス登録をリクエスト」

## 技術メモ(Claude向け)
- プロジェクトフォルダは日本語パス(`Desktop/プロジェクト一式/claude`)。Bashからも直接アクセス可能
- バイナリ(画像等)はClaudeが直接書き込めないため、Python(Pillow/pypdfium2)経由で生成・保存している
  - docx/PDFのページを画像化する際は、このMacに `soffice`(LibreOffice)や`pandoc`、`pdftoppm`が
    入っていないため、`pip install --user pypdfium2` でPDF→画像変換を行う(poppler不要)
  - docxは直接PDF変換する手段がないため、Word(Microsoft Office)で手動でPDFエクスポートしてもらい、
    それをpypdfium2で画像化する運用(Pages/Googleドキュメントはテキストボックスのレイアウトが崩れるため不可)
- 画像追加時は拡張子の大文字小文字に注意(例: image11.JPG)
- プレビュー機能(preview_start)でこのフォルダを直接サーブしようとすると権限エラーになることがある。
  その場合はスクラッチパッド配下にサイトをコピーしてそちらをサーブすると回避できる
- git操作でトークン等の機密情報をチャットやコマンド出力に表示しないよう注意(redactして扱うこと)
